WebSocket

Ein bidirektionales Netzwerkprotokoll für das Web

Ablauf

  • Dauer ca. 60 Minuten
  • Teil 1: Erklärung, Theorie
  • Teil 2: Demo/Code

Was ist WebSocket?

  • Netzwerkprotokoll basierend auf TCP
  • Initiiert über HTTP (im Browser via Javascript)
  • Bidirektionale Verbindung (full-duplex)
  • Teil der HTML5 Initiative
  • guter Browser Support

Was ist WebSocket NICHT ?

  • Eine Implementierung
  • Eine Alternative zu HTTP oder REST
  • Mobile-Only

Eigenschaften

  • Standardisiert (RFC 6455, Dez 2011)
  • Baut auf TCP auf (nicht HTTP)
  • Effizient (6 Bytes Header)
  • Bidirektional
  • Weit verbreitet
  • Gut unterstützt (Brower, Libraries, Webserver, ...)

Motivation

  • Einschränkungen Request+Response von HTTP zu umgehen
  • Ablöse von "Hacks" durch Long-Polling
  • Browser als Clientplatform der Zukunft stärken

Wozu braucht man das?

  • Für Server-Push
  • Für Datenübertragung mit niedriger Latenz (Echtzeitkommunikation)
  • Bei Flexibilität beim Design eines Protokolls

Javascript (Client)


var ws = new WebSocket('wss://example.com/socket'); // 1

ws.onerror = function (error) { ... } // 2
ws.onclose = function () { ... } // 3

ws.onopen = function () { // 4
  ws.send("Connection established. Hello server!"); // 5
}

ws.onmessage = function(msg) { // 6
  if(msg.data instanceof Blob) { // 7
    processBlob(msg.data);
  } else {
    processText(msg.data);
  }
}
